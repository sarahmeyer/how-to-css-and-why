          .header .nav .nav-right .item-group .nav-item .button {
            color: blue;
            margin-right: 1rem;
          }

If you wouldn't write this:

          .header .nav .nav-right .item-group .nav-item .button {
            color: blue;
            margin-right: 1rem;
          }

Then don't write this!

          .header {
            .nav {
              .nav-right {
                .item-group {
                  .nav-item {
                    .button {
                      color: blue;

                    }

CSS Specificity in 1 Slide

CSS Specificity in 1 Slide

  1. Type selectors (e.g., h1) and pseudo-elements (e.g., :before).
  2. Class selectors (e.g., .example), attributes selectors (e.g., [type="radio"]) and pseudo-classes (e.g., :hover).
  3. ID selectors (e.g., #example).

Source: MDN

I lied, it totally takes 2 slides.

Universal selector (*), combinators (+, >, ~, ' ') and negation pseudo-class (:not()) have no effect on specificity. (The selectors declared inside :not() do, however.)

Inline styles added to an element (e.g., style="font-weight:bold") always overwrite any styles in external stylesheets, and thus can be thought of as having the highest specificity.

When an !important rule is used on a style declaration, this declaration overrides any other declarations.

Recommendations

  1. BEM (CEM) syntax
  2. !important is a code smell
  3. Pay attention to the depth of SCSS nesting
  4. One SCSS file per template. side-bar/side-bar.html should have a corresponding SCSS file in side-bar/side-bar.scss.
